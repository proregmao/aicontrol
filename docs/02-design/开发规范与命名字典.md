## 📚 命名规范总则

### Go后端命名规范
```go
// 包名: 小写，简洁，描述性
package temperature
package server
package breaker

// 结构体: 大驼峰命名 (PascalCase)
type TemperatureSensor struct {}
type ServerInfo struct {}
type BreakerStatus struct {}

// 接口: 大驼峰命名，通常以 -er 结尾
type TemperatureReader interface {}
type ServerController interface {}

// 函数/方法: 大驼峰命名 (公开) 或小驼峰命名 (私有)
func GetTemperatureData() {}
func createConnection() {}

// 变量: 小驼峰命名
var temperatureData []float64
var serverList []*ServerInfo

// 常量: 大写，下划线分隔
const MAX_TEMPERATURE = 50.0
const DEFAULT_TIMEOUT = 30
```

### TypeScript前端命名规范
```typescript

type ServerStatus = 'online' | 'offline' | 'error'

let currentServer: ServerInfo | null = null

// 常量: 大写，下划线分隔
const API_BASE_URL = '/api/v1'
const MAX_RETRY_COUNT = 3
```
