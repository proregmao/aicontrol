🌡️ 温度单位修复验证

移除温度单位选择器，默认使用"°C"

🔧 修复说明

用户反馈: 温度的单位不需要选择，默认是度

修复内容:

  • 移除温度条件中的单位选择下拉框
  • 默认使用"°C"作为温度单位
  • 在界面上显示固定的"°C"标识
  • 简化用户操作流程

📊 修复前后对比

❌ 修复前

温度条件 1 - 前进风口 大于 > 25

问题:

  • 用户需要额外选择单位
  • 界面元素过多
  • 操作步骤复杂

✅ 修复后

温度条件 1 - 前进风口 大于 > 25 °C

优势:

  • 默认使用°C，无需选择
  • 界面更简洁
  • 操作更便捷

🔧 技术实现细节

1. 移除单位选择器

2. 默认单位设置

const addCondition = () => {
  const condition = {
    id: Date.now().toString(),
    type: '',
    operator: '',
    value: '',
    unit: '°C',  // 默认设置为°C
    sensorId: '',
    sensorName: '',
    timeValue: null
  }
  strategyForm.value.conditions.push(condition)
}

3. 显示文本优化

const getConditionText = (condition: any) => {
  if (condition.type === 'temperature') {
    const sensorName = temperatureSensors.value.find(s => s.id === condition.sensorId)?.name || '温度传感器'
    const unit = condition.unit || '°C'  // 默认使用°C
    return `${sensorName} ${condition.operator} ${condition.value}${unit}`
  }
  // ...
}

🎯 用户体验改进

简化的操作流程：

  1. 选择条件类型：温度条件
  2. 选择温度探头：1 - 前进风口
  3. 选择比较符：大于 >
  4. 输入温度值：25
  5. 单位自动显示：°C (无需选择)

结果显示： "前进风口 > 25°C"
